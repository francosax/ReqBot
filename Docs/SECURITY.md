## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

If you believe you have found a security vulnerability in this project, please report it to us as described below.

### How to Report

Please email us at reqbotproject@gmail.com .

Subject: Security Vulnerability: [Short Description]

Body: Please include as much information as possible, including:

The type of issue (e.g., buffer overflow, SQL injection, XSS).

Full paths to the source file(s) related to the manifestation of the bug.

The location of the affected source code (tag/branch/commit or direct URL).

Any special configuration required to reproduce the issue.

Step-by-step instructions to reproduce the issue.

Proof-of-concept or exploit code (if possible).

### Our Response Process

Acknowledgment: We will respond to your report within 48 hours to acknowledge that we have received it.

Investigation: We will investigate the issue and determine its impact. We may contact you for further information.

Resolution: Once the issue is confirmed, we will work to release a patch or workaround.

Disclosure: We ask that you maintain confidentiality until we have released a fix. Once the fix is public, we will credit you for the discovery (unless you prefer to remain anonymous).

### Scope

The following are in scope:

Vulnerabilities in the code stored in this repository.

Vulnerabilities in the specific implementation of our API.

The following are out of scope:

Vulnerabilities in third-party dependencies (please report those to the respective maintainers).

DDoS attacks or spam.

Social engineering attacks.

Preferred Languages

We prefer all communications to be in English or Italian.

Thank you for helping keep our community safe.
